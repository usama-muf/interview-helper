## Internal Working of HashMap

`HashMap` in Java works on the principle of Hashing. It uses a data structure called an array of `Node` (or `Entry` in older versions) to store key-value pairs.

### Process of `put(key, value)`

1. **Hashing**: The `hashCode()` method of the key is called. The hash is then shifted and XORed to reduce collisions.
2. **Index Calculation**: `index = (n - 1) & hash`, where `n` is the capacity of the array.
3. **Collision Handling**:
   - If the calculated index is empty, the node is placed there.
   - If a node already exists (collision), it checks the key. If the key is the same, it replaces the value.
   - If the key is different, it adds the new node to the end of the Linked List (chaining).
   - **Java 8 Optimization**: Once the number of items in a bucket reaches a certain threshold (8), the Linked List is converted into a **Red-Black Tree** to improve worst-case search time from `O(n)` to `O(log n)`.

### Load Factor and Capacity

- Default initial capacity: **16**
- Default load factor: **0.75**
- When the number of entries exceeds the capacity multiplied by the load factor, the HashMap is **resized** (rehashing occurs), typically doubling the array size.
